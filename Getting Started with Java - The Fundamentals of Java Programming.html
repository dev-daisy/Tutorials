Getting Started with Java: The Fundamentals of Java Programming

This course serves as a theoretical introduction to the Java programming language - a class-based, object-oriented, and platform-independent language. Use this course to discover what these qualities mean when using Java to develop programs. Explore some of the significant features of Java. Get a glimpse into the Java ecosystem and the phases involved in translating Java code to a machine-executed form. In doing this, you'll examine concepts such as compilers, Java bytecode, and the Java Virtual Machine. Delve into some of the other properties of the language, such as using the just-in-time compiler and the automatic garbage collector. Finally, learn about the vast array of fields in which Java programming can be applied. At the end of this course, you'll have a solid foundational comprehension of the characteristics and use cases of the Java programming language.

What this particular tool does is to periodically clean up unused objects in your program.

If you have any prior programming experience, you will know that you may create some objects or variables somewhere in the middle of a program, and then beyond that, you don't really need it. Well, what about the resources which are consumed by those objects? Wouldn't it make sense to free up those resources and make them available to other objects which might be required by your program as well as others on your system? Well, the answer is yes, and this is precisely what Java's garbage collector does.

So this is an independent process which scans the heap memory and then is able to identify objects which are no longer needed by a program. Such objects are categorized as unused objects, and in the context of the Java garbage collector, an unused object is one which is no longer referenced by any part of a running program. So, if you're halfway through running your program, and in the half which still needs to be run, there are no more references to certain objects which are created in the first half. These are marked as unreferenced objects and the garbage collector will delete them and while doing so, will free up the resources such as memory and processor time which is consumed by them.

This has the effect of having programs run more efficiently since they'll have more resources to work with. Significantly, it prevents programmers from explicitly having to free up their resources once they're no longer required. C programmers will be aware that in order to allocate resources for a particular object, we need to call the malloc function, and once that object is no longer needed we need to call the free function. All of this is not required in Java since the garbage collector takes care of it for us.

That said however, the garbage collection does add some additional overhead. That is, since it is a separate process which runs in the background it does consume some resources of its own and in many instances, it is not as efficient as having developers explicitly allocate and free up resources. That said however, more often than not, the pros of having automatic garbage collection outweigh the costs associated with it.
